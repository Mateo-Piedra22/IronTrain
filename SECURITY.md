# Security Policy

## Supported Versions

IronTrain currently supports security updates for the latest version on the default branch (`master`).

| Version | Supported |
| ------- | --------- |
| `master`  | ✅ |
| older releases/tags | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Use one of these private channels:

1. GitHub Security Advisories (preferred):
   - Go to the repository **Security** tab
   - Click **Report a vulnerability**
2. If advisory flow is unavailable, contact the maintainer directly.

Please include:

- Affected component(s)
- Reproduction steps / proof of concept
- Impact assessment
- Suggested remediation (if available)

## Response SLA

- **Acknowledgement**: within **24 hours**
- **Initial triage**: within **3 business days**
- **Remediation plan**: within **7 calendar days** for confirmed high/critical issues

## Disclosure Policy

- We coordinate disclosure after a fix is available.
- Credit can be provided to reporters unless anonymity is requested.

## Scope

In scope:

- Source code in this repository
- CI/CD workflows and build/release pipeline
- Mobile/web app dependencies and configuration

Out of scope:

- Social engineering/phishing attempts
- Denial-of-service testing against production services
- Reports without actionable reproduction steps
